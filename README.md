# Go2 IsaacLab Gazebo ROS 2 Workspace

This repository is a ROS 2 and Gazebo Classic workspace for deploying and testing a Unitree Go2 reinforcement learning policy before moving toward real robot use.

The main job of this repo is **sim-to-sim deployment**:

```text
IsaacLab trains the policy
    -> exported policy.pt is copied here
    -> rl_sar builds the policy observation from Gazebo/ROS state
    -> the policy outputs joint-position actions
    -> robot_joint_controller converts commands into Gazebo joint efforts
    -> Gazebo simulates the Go2
```

In simple terms: IsaacLab is where the robot learns. This repo is where we check whether the learned policy still works in Gazebo with ROS 2.

That IsaacLab task trains the policy with weak or broken actuators using:

```text
tau_actual = alpha * tau_command
```

where `alpha = 1.0` means healthy and lower values mean weak or broken joints.

| Shared Go2 config | `policy/go2/base.yaml` | Base Go2 timing/gains/defaults |

## Runtime Flow

```text
Gazebo Go2
    -> joint states and IMU
    -> robot_joint_controller/state and /imu
    -> rl_sim
    -> RLSDK observation vector
    -> policy/go2/isaaclab48/policy.pt
    -> desired joint positions
    -> robot_joint_controller/command
    -> PD/effort commands
    -> Gazebo joints
```

The most important sim-to-sim contract is that Gazebo must send the policy the same style of observation that IsaacLab used during training.

Current policy contract:

```text
observation size: 48
action size: 12
action type: desired joint position
policy folder: policy/go2/isaaclab48
```

## Build

From this repo:

```bash
cd ~/Projects/go2_isaac_gazebo
source /opt/ros/humble/setup.bash
./build.sh robot_msgs robot_joint_controller go2_description rl_sar
source install/setup.bash
```

The helper script creates the correct ROS 2 `package.xml` symlinks, checks/downloads runtime assets, then runs `colcon build`.

Rebuild after changing C++ or xacro files, for example:

```text
src/rl_sar/src/rl_sim.cpp
src/rl_sar/fsm_robot/fsm_go2.hpp
src/rl_sar/library/core/rl_sdk/rl_sdk.cpp
src/robot_joint_controller/ros2/src/robot_joint_controller_group.cpp
src/rl_sar_zoo/go2_description/xacro/*.xacro
```

For policy `.pt` or YAML-only changes, you usually only need to restart `rl_sim`.

## Keyboard Control

Use the terminal running `rl_sim`.

```text
0      get up into default standing pose
1      enter RL policy controller
W/S    increase/decrease forward command
A/D    increase lateral-left/lateral-right command
Q/E    increase yaw-left/yaw-right command
Space  reset x/y/yaw command to zero
R      reset Gazebo world
Enter  pause/unpause simulation
```

Recommended startup sequence:

```text
0
wait for get-up to finish
1
confirm: RL Controller [isaaclab48] holding default pose
W
```

## Policy Management

To replace the deployed policy:

1. Export the trained policy from IsaacLab as TorchScript.
2. Copy it to:

```text
policy/go2/isaaclab48/policy.pt
```

3. Update this config to match training:

```text
policy/go2/isaaclab48/config.yaml
```

Check these fields carefully:

```text
num_observations
observations
action_scale
default_dof_pos
joint_mapping
rl_kp / rl_kd
torque_limits
joint_pos_limit_lower / joint_pos_limit_upper
```

4. Restart `rl_sim`.

The active policy name is selected in:

```text
src/rl_sar/fsm_robot/fsm_go2.hpp
```

Current value:

```text
isaaclab48
```

## Common Debug Checklist

If the robot stands badly after pressing `1`:

```text
confirm terminal says "holding default pose"
check default_dof_pos
check joint_mapping
check observation size is 48
check the correct policy folder is loaded
```

If the robot walks in a curve:

```text
check yaw command is zero
check lateral command is zero
inspect CSV for left/right joint tracking mismatch
check calf targets are not constantly hitting joint limits
compare Gazebo joint order against IsaacLab joint order
```

If the robot falls immediately:

```text
restart Gazebo and rl_sim
press 0 and wait for get-up before pressing 1
confirm the policy.pt and config.yaml belong together
rebuild if C++ or xacro files changed
```

## Safety Notes

This repo is still for simulation and deployment testing. Passing Gazebo does not automatically mean the policy is safe for the real Go2.

Before real robot use, re-check:

```text
observation order
action scale
joint order
control frequency
PD gains
torque limits
joint limits
emergency stop behavior
fault behavior
```
